如题，这篇文章主要讲如何使用Hexo和Github Pages搭建个人博客。
各位想尝试的读者可以先看一遍，再实践。

注册Github账号

先打开官网

官网地址：https://github.com/

然后再点右上角的Sign Up按钮
或者直接进入注册页面：https://github.com/signup
有个邮箱就可以注册了。

这里推荐一个油猴脚本：Github汉化脚本

安装依赖

注：尽量将node、npm、git等添加到环境变量

安装node.js

官网地址：https://nodejs.org/zh-cn
下载页面：https://nodejs.org/zh-cn/download

根据需求安装即可
在cmd[1]中执行node -v和npm -v，正常输出版本号就说明安装成功

安装git

官网地址：https://git-scm.com/
下载页面：https://git-scm.com/downloads

同样，根据需求安装即可
在cmd中执行git -v，正常输出版本号就说明安装成功

官网地址：https://hexo.io/zh-cn/
官方文档：https://hexo.io/zh-cn/docs/

这里没有贴上下载地址是因为这个需要用npm安装
官网已经把命令贴上了：

这里我来详细讲下每行命令的作用

第一行npm install hexo-cli -g安装了Hexo的基础环境。
第二行hexo init blog会在当前工作目录创建一个blog文件夹，并对其初始化。
第三行cd blog是命令提示符基础命令，将工作目录切换到刚创建的blog文件夹。
第四行npm install作用是安装。
第五行hexo server作用是启动服务器

自定义博客

自定义主题

hexo自带的主题或许不美观，这时，就需要自定义主题
hexo官网上有大量的博客主题可供使用，我的这个博客的主题是jelly，如果你想要别的主题也可以，请访问https://hexo.io/themes

每个博客主题都会提供安装教程，跟着你选择的主题的教程走即可，这里以我选的jelly主题为例，读https://github.com/preccrep/hexo-theme-jelly/blob/main/README.md就行

插件

官方也列出了非常多的插件，还是那句话，根据需求安装即可。

管理面板

这里我要专门讲一个叫Bridge的插件。

这里贴一下地址：https://github.com/DeepSpaceHarbor/hexo-bridge
在上面讲到的插件的页面也能找到这个插件。

在你的博客的根目录执行npm i --save hexo-bridge命令去安装这个插件。
然后再去执行hexo server

这时打开http://localhost:4000/bridge/ 就可以看到管理面板了，只是都是英文的，英语不好的可以在浏览器启用翻译功能，这对不熟悉和懒得用命令的博主极其友好。

这里讲了这个插件就不讲那些命令了。

使用Github Pages

基础

浏览器中访问https://github.com ，登录你注册的账号（已登录的跳过），创建一个叫你的用户名.github.io的仓库，必须是这个，其他的都不行。

然后点*设置（Settings）在打开的页面左边栏找到Github Pages（Pages）*根据需求设置就行。

发布博客

发布到Github Pages

在前面的教程中，已经使你的博客已经可以在本地（http://localhost:4000）查看，但这是远远不够的，我们需要让其他人看到，前面讲到的Github Pages就可以让你的网页被互联网上的其他人看到

我们如果要让我们的博客被别人看到，就可以把博客发布到Github Pages

怎么做呢？

先通过在博客根目录执行npm install hexo-deployer-git --save安装部署器
然后，执行hexo clean清除多余的内容，最后执行hexo g && hexo d生成静态文件并部署

过一段时间，在浏览器打开http://*你的用户名*.github.io/如果成功地显示的博客内容则说明博客搭建成功

更新以同样的方式，将新的内容重新部署即可

详情请见：官方文档

当你无法部署时，可能部署器损坏，重新执行npm install hexo-deployer-git --save安装即可。

解决Github访问慢

由于某些原因，Github Pages在国内访问较慢，因此，我们需要另外想办法。
我们可以尝试托管到其他平台，比如Vercel、Netlify或Render。

我这边是用了Render。

先到官网连接到你的Github账户，然后到仪表板新建，选择你的仓库就行了。
